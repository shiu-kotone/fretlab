# FretLab — オールインワン・ギター練習アプリ 詳細仕様書

- バージョン: 1.0
- 作成日: 2026-07-02
- 想定読者: 実装を担当するAIエージェント(Claude Code)および発注者
- アプリ名は仮称。実装時に変更可。

---

## 1. プロジェクト概要

### 1.1 目的
ギター練習に必要な機能を1つに統合したアプリを作る。指板演奏・コードライブラリ・スケール表示・メトロノーム・チューナー・コード進行再生・録音・練習記録の8機能を、単体で完結するオフライン動作可能なアプリとして提供する。

### 1.2 ターゲットユーザー
- 初心者〜中級者のギタリスト(アコギ/エレキ両方)
- 発注者本人がメインユーザー。左利き対応は必須要件
- 日本語UIを第一言語とする

### 1.3 成功基準
- iPhoneのホーム画面から起動し、機内モードでも全機能(チューナーのマイク・録音を除く)が動作する
- チューナーの検出精度が±3セント以内、メトロノームの拍間ジッターが±3ms以内
- 上記8機能すべてが左利き設定・カスタムチューニング設定と矛盾なく連動する

## 2. 技術方針(重要な前提)

### 2.1 なぜPWAか
発注者の開発環境はiPhoneのみ(Mac/Xcodeなし)。ネイティブiOSアプリはビルド・署名・インストールにXcodeが必須のため選択できない。そのため本アプリは **PWA (Progressive Web App)** として実装する。

- Safariで開き「ホーム画面に追加」するとフルスクリーンのスタンドアロンアプリとして起動する
- Service Workerにより完全オフラインで動作する
- Web Audio APIでリアルタイム音声合成、getUserMediaでマイク入力、MediaRecorderで録音、IndexedDBでデータ永続化が可能であり、要求機能はすべて実現できる
- 開発はClaude Code on the web(claude.ai/code)+ GitHubリポジトリで行い、GitHub Pagesへ自動デプロイする。iPhoneだけで開発サイクルが完結する

### 2.2 技術スタック(指定)
| 項目 | 採用技術 | 備考 |
|---|---|---|
| ビルド | Vite + TypeScript (strict) | |
| UI | React 18+ (関数コンポーネント + Hooks) | |
| 状態管理 | Zustand (persistミドルウェアで設定永続化) | Reduxは使わない |
| スタイル | CSSモジュール or vanilla CSS + CSS変数 | 重いUIフレームワーク禁止。Tailwindは任意 |
| 音声 | **素のWeb Audio API**(自前エンジン) | Tone.js等の外部音声ライブラリは使わない(精度とサイズのため) |
| ピッチ検出 | 自前実装(YINアルゴリズム) | 外部ライブラリ可(pitchy等)だが精度要件を満たすこと |
| 永続化 | 設定=localStorage(Zustand persist)、録音・練習記録・ユーザー作成データ=IndexedDB(Dexie.js) | |
| PWA | vite-plugin-pwa (Workbox) | precacheで全アセットをオフライン化 |
| デプロイ | GitHub Actions → GitHub Pages | mainへのpushで自動デプロイ。`vite.config`の`base`をリポジトリ名に設定 |
| テスト | Vitest(音楽理論ロジック・ピッチ検出・スケジューラは必須) | UIテストは任意 |

### 2.3 リポジトリ構成(推奨)
```
/src
  /audio        # AudioEngine, scheduler, synth(pluck/click), pitch-detect
  /theory       # 音楽理論: note, interval, chord, scale, tuning(純粋関数のみ・UI非依存)
  /data         # コードボイシングDB, スケール定義, プリセット進行
  /components   # 共有UIコンポーネント(Fretboard, ChordDiagram, Knob等)
  /features
    /fretboard  /chords  /scales  /metronome  /tuner
    /progression  /recorder  /log  /settings
  /stores       # Zustandストア
  /pwa
/tests
/docs           # 本仕様書のコピー、ADR
```

**設計原則**: `theory`と`audio`はReactに依存しない純粋TypeScriptモジュールにする。将来SwiftUIへ移植する際、この層のロジック仕様(テスト)がそのまま移植仕様書になる。

## 4. 共通仕様(全機能に横断)

### 4.1 左利きモード(グローバル設定)
- 設定 > 「左利きモード」ON/OFF。既定OFF
- ONの場合:
  - 指板表示: 左右ミラー(ナットが右、高フレットが左)
  - コードダイアグラム: 左右ミラー(1弦と6弦の位置が入れ替わる)
  - スケール表示・度数ハイライト・進行プレイヤー内のダイアグラムもすべて連動
- 実装は「描画時に座標変換する」一箇所で吸収し、データ(フレット番号等)は反転しない

### 4.2 チューニング(グローバル設定)
- アプリ全体で1つの「現在のチューニング」を共有する。指板の発音・音名表示、チューナーの目標音、コードライブラリの注記がすべてこれに従う
- プリセット: レギュラー(EADGBE)、半音下げ、全音下げ、Drop D、Drop C、DADGAD、Open G、Open D
- カスタム: 各弦を半音単位で±5半音まで個別変更し、名前を付けて保存(IndexedDB)。削除可
- **注意**: コードライブラリのボイシングDBはレギュラーチューニング前提。レギュラー以外選択中はコードタブに「ダイアグラムはレギュラー基準」の注記バッジを出す(押さえ方の自動再計算はスコープ外)
- 基準ピッチ A4 = 440Hz 既定。415〜466Hzで1Hz刻み変更可(チューナー・発音の両方に反映)

### 4.3 音名表記(グローバル設定)
- 英名(C, C#…) / ドレミ(ド, ド#…) 切替
- ♯表記 / ♭表記 切替(コード名は文脈優先: 例 F♯m は設定に関わらず一般的表記を使用してよい)

### 4.4 オーディオエンジン(全機能共通・最重要モジュール)
単一の`AudioContext`をアプリ全体で共有する。

- **アンロック**: iOS Safariでは初回のユーザー操作(タッチ)まで`AudioContext`がsuspended。最初のタップで`resume()`する共通処理を入れる。suspended中に発音操作をした場合は自動resume→発音
- **サイレントスイッチ対策**: 対応環境では `navigator.audioSession.type = 'playback'` を設定し、消音スイッチONでも音が出るようにする(未対応環境では諦め、初回にトースト等で案内)
- **スケジューラ**: メトロノーム・進行再生は `setInterval`(25ms) のlookaheadループで `AudioContext.currentTime` 基準の絶対時刻に **100ms先読み** でイベントを予約する方式(いわゆる "A Tale of Two Clocks" 方式)。`setTimeout`で直接鳴らす実装は禁止(ジッターが出る)
- **ギター音合成**: Karplus-Strong法(ノイズバースト+遅延線+ローパスフィードバック)で弦の音を合成する。サンプル音源は同梱しない(サイズとライセンスの都合)。パラメータ: 減衰(サステイン約1.5〜3秒)、ブライトネス、ベロシティ。ポリフォニー最大12ボイス、超過時は最古ボイスを解放
- **クリック音合成**: 種類ごとに合成(§5.4 メトロノーム)
- **マスターボリューム**: 設定でギター音/クリック音を個別に0〜100%
- バックグラウンド制約: 画面ロック中の継続再生は保証しない。代わりに再生中は Screen Wake Lock API で画面消灯を防ぐ(設定でOFF可)

### 4.5 発音の衝突ルール
- メトロノームと進行再生は同時に起動しない(片方を開始したらもう片方を停止)
- 指板タップ音・コード試聴はメトロノーム/進行再生と共存可
- チューナー(マイク)使用中もメトロノームは併用可(ヘッドホン推奨の注記を表示)

### 4.6 テーマ・デザイン指針
- 既定はダークテーマ(ステージ/アンプの世界観)。ライトテーマも用意し設定で切替、既定は端末設定に追従
- デザイントークン(実装時にCSS変数化):
  - `--bg: #16130F`(消灯したステージの黒に近い暖黒) / `--surface: #221D17` / `--line: #3A322A`
  - `--accent: #E8A33D`(真空管アンプのアンバーグロー。再生中インジケータ・アクティブ状態に使用)
  - `--string: #C8CDD4`(ニッケル弦のシルバー。指板の弦・目盛り)
  - `--ok: #6FBF73`(チューナー合致) / `--warn: #E0564B`
  - 度数カラー(指板・ダイアグラム共通): ルート=`--accent`、3度系=`#D96C5F`、5度=`#5F9ED9`、7度系=`#9C7BD9`、その他テンション=`#7A8794`
- タイポグラフィ: 見出し・BPM等の大型数字は幾何学的サンセリフ(例: Archivo/Bricolage Grotesque をサブセットして同梱)+ 数値表示は等幅数字(`font-variant-numeric: tabular-nums`)。本文はシステムフォント(`-apple-system`)
- シグネチャ要素は「指板」そのもの: 弦は太さを1〜6弦で変える・巻弦(4-6弦)は質感を変える等、指板の描画品質に最も力を入れる。他のUIは静かに保つ
- 過剰装飾・過剰アニメーション禁止。拍のフラッシュなど機能的モーションのみ。`prefers-reduced-motion`尊重
- タップ対象は最小44×44pt。ボタン文言は動詞で具体的に(「保存」「再生」)

### 4.7 データ保全
- 設定エクスポート/インポート: 練習記録・カスタムチューニング・自作進行・お気に入りをJSON1ファイルで書き出し/読み込み(録音音声は含めない。録音は個別に共有シートで書き出し)
- IndexedDBの使用容量と録音合計サイズを設定画面に表示

## 5. 機能別 詳細仕様

## 5.1 指板演奏モード(タブ: 指板)

### 目的
指板をタップして音を確かめ、音名・度数・スケールの位置関係を視覚と耳で学ぶ。

### 表示
- 6弦 × 0〜22フレットの指板を横向きに描画(SVGまたはCanvas。ズーム時の描画品質からSVG推奨)
- 既定表示範囲は0〜12フレット。横スクロールで22フレットまで。ピンチまたは「12F/15F/22F」切替ボタンでズーム
- ポジションマーク: 3,5,7,9,15,17,19,21にドット、12にダブルドット
- 弦の向き: 既定「1弦(高音)が上」= プレイヤー視点。設定で「6弦が上」(譜面/ダイアグラム視点)に切替可
- 左利きモードでミラー表示(§4.1)
- 各フレット位置に音名ラベル(表示設定: なし / すべて / ナチュラル音のみ / タップした音のみ。既定=ナチュラル音のみ)
- 現在のチューニング(§4.2)に完全連動。開放弦名を指板左端(左利き時は右端)に表示

### 操作と発音
- タップ: その位置の音をKarplus-Strong音で発音。開放弦はフレット0領域(ナット左)をタップ
- マルチタッチ対応(最大6点同時=コードが押さえられる)
- 同一弦上をタッチしたまま横に滑らせたら発音位置を更新(簡易スライド。ピッチベンドは不要、離散的に鳴らし直す)
- 発音中の位置はアンバーに点灯し減衰に合わせてフェード

### 度数ハイライト機能(要件)
- ツールバーの「度数」ボタンでONにすると度数モードに入る:
  1. 最初のタップでその音を**ルート(R)**に設定(発音もする)
  2. 画面下の度数チップ(R, ♭2, 2, ♭3, 3, 4, ♭5, 5, ♯5, 6, ♭7, 7)から任意の複数を選択
  3. 選択した度数に該当する全ポジションが度数カラー(§4.6)でハイライトされる
- プリセットチップ: メジャートライアド(R,3,5) / マイナートライアド(R,♭3,5) / 7th(R,3,5,♭7) / maj7 / m7 / パワーコード(R,5)
- ルートは長押しでいつでも変更可。「クリア」で解除

### スケールオーバーレイ(追加要件: スケール/度数の指板表示)
- ツールバーの「スケール」ボタンでパネルを開く: ルート(12音) × スケール選択
- 収録スケール(最低限):
  - メジャー(イオニアン)、ナチュラルマイナー、ハーモニックマイナー、メロディックマイナー
  - メジャーペンタトニック、マイナーペンタトニック、ブルーススケール
  - チャーチモード7種(ドリアン、フリジアン、リディアン、ミクソリディアン、エオリアン、ロクリアン ※イオニアン含む)
  - ディミニッシュ(H-W / W-H)、ホールトーン、オルタード、リディアン♭7、ミクソリディアン♭9♭13、ハーモニックマイナーP5below(=フリジアンドミナント)
- 表示: 構成音を指板全体にドット表示。ラベルは「度数 / 音名」切替。ルートは常に強調
- ポジション絞り込み: 「全体 / ボックス1〜5(ペンタ・CAGED準拠の5ポジション)」。ボックス選択時は該当フレット範囲外を減光
- スケール表示中も指板タップで発音可(スケール外の音も鳴らせるが、スケール外タップは灰色フラッシュで区別)
- スケール定義はインターバル配列で `theory/scales.ts` に一元管理(例: メジャー=[0,2,4,5,7,9,11])

### 状態保持
最後に使ったスケール・度数選択・表示範囲はセッションをまたいで復元する。

## 5.2 コードライブラリ(タブ: コード > 「ライブラリ」)

### 目的
あらゆるコードの押さえ方(ローコードに限らず複数ポジション)を参照し、音で確認する。

### コード網羅範囲
- ルート: 12音(異名同音は表記設定に従い片方を表示、検索は両方ヒット)
- タイプ(最低26種): maj, m, 5(パワーコード), dim, aug, sus2, sus4, 6, m6, 69, maj7, m7, 7, mMaj7, dim7, m7♭5, maj9, m9, 9, add9, madd9, 11, m11, 13, 7sus4, 7♭9, 7♯9, 7♯5, 7♭5, maj7♯11
- 合計 12 × 30種前後 × 各3〜6ボイシング

### ボイシングデータ設計(重要)
全ボイシングを手書きせず、**「オープンコードは手書きDB + ムーバブルフォームはテンプレート生成」**のハイブリッドで実装する:
1. `data/openChords.ts`: 一般的なオープンコード(C, A, G, E, D系とその変化形 約80〜120個)を手書き定義
2. `data/movableForms.ts`: CAGED各型のムーバブルフォーム(E型バレー、A型バレー、C型、G型、D型)と、トライアド型(1-3弦 / 2-4弦 / 3-5弦)、シェルボイシング(R-3-7: 6弦ルート/5弦ルート)を**コードタイプごとに**テンプレート定義し、ルートに応じてフレット位置を計算して全12キーに展開
3. 生成結果は起動時に合成し、各コードにつきポジション順(ローポジション→ハイポジション)に並べる
- **検証必須**: テンプレート展開後の全ボイシングについて「押さえた音の集合がコードの構成音集合と一致するか」を自動テストで全件検証する(theory層の`chordTones()`と突合)。1件でも不一致があれば生成ロジックのバグ

```ts
// ボイシングの型(6弦→1弦の順)
interface Voicing {
  frets: (number | 'x')[];   // 例 C: ['x',3,2,0,1,0]
  fingers?: (0|1|2|3|4)[];   // 運指(0=開放)
  barres?: { fret: number; fromString: number; toString: number }[];
  baseFret: number;          // ダイアグラム左端(=1なら開放位置表示)
  label: 'オープン'|'E型'|'A型'|'C型'|'G型'|'D型'|'トライアド'|'シェル';
}
```

### 画面構成
- 上段: ルート選択(12個の横スクロールチップ) + タイプ選択(グループ化されたリスト: 基本 / セブンス / テンション / その他)
- テキスト検索バー: "C#m7b5" "F♯m7♭5" "Dbmaj7" 等の表記ゆれを正規化してヒット
- 中央: 選択コードのダイアグラムを大きく1枚表示、左右スワイプ(またはドット)でボイシング切替
- ダイアグラム仕様:
  - 縦型(弦が縦線・フレットが横線)。ナット表示 or baseFret数字表示
  - ミュート弦=×、開放弦=○、押弦=指番号入りドット、バレー=アーチ
  - 各弦の下に実音名と度数(R/3/♭7等)を表示
  - 左利きモードで左右ミラー
- 再生: 「ストラム」ボタン(6弦→1弦へ各30msずらし発音、左利き設定でも音順は同じ)/ 長押しで「アルペジオ」(1音ずつ120ms間隔)
- お気に入り(★): コード+ボイシング単位で保存、ライブラリ上部の「★」フィルタで一覧
- 「指板で見る」ボタン: 指板タブへ遷移し、そのコードの構成音を度数ハイライトとして表示

## 5.3 コード進行プレイヤー(タブ: コード > 「進行」)

### 目的
コード進行を鳴らしてバッキングにし、その上でスケール練習・アドリブ練習をする。

### データモデル
```ts
interface Progression {
  id: string; name: string;
  bpm: number;                 // 40–240
  timeSig: { beats: number; unit: 4|8 };  // 4/4, 3/4, 6/8, 12/8
  strumPatternId: string;
  bars: Bar[];                 // 1–64小節
  loop: boolean;
  createdAt: number; updatedAt: number;
}
interface Bar { chords: { chordId: string; voicingIndex: number; beats: number }[] } // 小節内1–2個、beats合計=拍子
```

### 編集UI
- 小節グリッド(1行4小節)。小節タップ→コードピッカー(ライブラリと同じルート×タイプ選択+ボイシング選択)
- 小節の分割(前半/後半で2コード)、複製、削除、ドラッグ並べ替え
- 移調ボタン: 全体を±半音単位でトランスポーズ(ボイシングは同型を平行移動、オープン系は最寄りの型に置換)

### ストラムパターン(プリセット)
全音符 / 2分 / 4分ダウン / 8分オルタネート(DUDU) / 定番「D–DU–UDU」/ 16ビート / シャッフル / アルペジオ3種(6弦から順・パターンA/B)。パターンはステップ列(タイミング+D/U/休符+アクセント)として`data/strumPatterns.ts`に定義。

### 再生
- §4.4の共通スケジューラ上で駆動。カウントイン1小節(クリック音)→ループ再生
- クリック併走ON/OFF、クリック音量個別
- 再生中表示: コード譜ビュー(現在小節をアンバーでハイライト)+ 画面下部に「現在のコード / 次のコード」のダイアグラム2枚(左利き連動)
- BPMは再生中も変更可(次拍から反映)

### プリセット進行(初期同梱)
12小節ブルース(A)、カノン進行(C)、王道進行 IV△7–V7–iii7–vi(C)、小室進行 vi–IV–V–I(C)、2‑5‑1(C)、50s進行 I–vi–IV–V(C)。ユーザー作成分はIndexedDBに保存(複製して編集可)。

## 5.4 メトロノーム(タブ: リズム)

### 基本パラメータ
| 項目 | 範囲/選択肢 | 既定 |
|---|---|---|
| BPM | 20–300(1刻み。±ボタン長押しで加速、大型ダイヤル/スライダー併設) | 100 |
| タップテンポ | 専用ボタン。直近4タップの平均、2.5秒無操作でリセット | — |
| 拍子 | 分子1–12 × 分母2/4/8/16。クイック選択: 2/4,3/4,4/4,5/4,6/8,7/8,12/8 | 4/4 |
| 音価(サブディビジョン) | 4分 / 8分 / 3連 / 16分 / シャッフル(スウィング8分) | 4分 |
| サブ音量 | 表拍に対する裏拍・細分音の音量 0–100% | 60% |
| 音色 | §下記5種 | ウッドブロック |

### 音色(すべてWeb Audioで合成。サンプル不使用)
1. **ウッドブロック**: 短い正弦+三角のバンドパス、ピッチ違いでアクセント(強拍1600Hz/弱拍1100Hz/細分800Hz)
2. **クリック(電子)**: 矩形波の極短パルス
3. **ビープ**: 正弦波50ms、強拍は5度上
4. **カウベル**: 2つの矩形波(545Hz+800Hz)+短い減衰
5. **リムショット風**: ノイズバースト+ハイパス

### アクセント編集
- 画面上部に拍ドットを1小節分表示。各ドットはタップで「アクセント → 通常 → ミュート」を循環
- 再生中は現在拍のドットが拡大+アンバーフラッシュ。設定で「全画面フラッシュ」も選択可(視覚メトロノーム)

### トレーナー機能
- **スピードトレーナー**: ON時「N小節ごとに +M BPM、上限X BPM」を設定(N:1–16, M:1–20, X≦300)。到達後は維持
- **ミュート小節**: 「Aマ小節鳴らして B小節ミュート」(A,B: 1–8)。体内メトロノーム訓練用
- 両者は併用可

### 精度要件
- lookaheadスケジューラ必須(§4.4)。拍間隔の実測ジッター±3ms以内(AudioContext時刻基準)
- 再生中はWake Lockで画面消灯防止(設定でOFF可)
- 再生状態はタブ移動しても継続。タブバーのリズムアイコンに再生中バッジ

---

## 5.5 チューナー(タブ: チューナー)

### モード1: マイクチューナー(既定)
- 初回に`getUserMedia({ audio: { echoCancellation:false, noiseSuppression:false, autoGainControl:false } })`で許可取得。拒否時は案内画面(設定アプリでの再許可手順)を表示し、モード2(音出し)は使用可のままにする
- **ピッチ検出**: YINアルゴリズム(閾値0.1〜0.15)。サンプルバッファ4096、検出範囲 E1(41Hz)〜E6(1319Hz)。信頼度が低いフレームは表示を更新しない(ちらつき防止)。直近5フレームの中央値でスムージング
- **表示**:
  - 中央に検出音名(大)+オクターブ、下に周波数(Hz, 小)
  - セントメーター: ±50セントの水平ゲージ+針。±5セント以内で針とリングが`--ok`緑に変化し、0.5秒維持でチェックマーク+短いハプティクス風フラッシュ
  - 現在のチューニング(§4.2)の6弦分の目標音チップを表示し、検出音に最も近い弦を自動ハイライト(オート弦判定)。チップをタップすると手動固定モード(その弦との差分だけを表示)
- 基準ピッチA4は§4.2の設定に従う。チューナー画面にも現在値(例: A=440)を表示しタップで変更可
- カスタムチューニング含む全プリセットに対応(目標音はチューニング定義から算出)

### モード2: 音出し(リファレンストーン)
- 6弦分のボタンを表示、タップでその弦の基準音をKarplus-Strong音で3秒再生(リピートトグルあり)
- マイク不許可環境・耳チューニング派のためのフォールバック

---

## 5.6 録音・再生(タブ: ラボ > 「録音」)

- 録音: MediaRecorderを使用。`MediaRecorder.isTypeSupported`で `audio/mp4` → `audio/webm` の順にフォールバック。録音中はレベルメーター+経過時間表示。最長30分/件
- 保存: 停止時に自動でIndexedDBへBlob保存。既定名「録音 YYYY-MM-DD HH:mm」、後からリネーム可
- 一覧: 日付降順。行に名前・日付・長さ・サイズ。スワイプまたはメニューで削除(確認ダイアログ必須)・共有(Web Share APIでファイル書き出し)
- 再生: シークバー、A-Bループ(区間リピート練習用)、再生速度 0.75/1.0倍
- メトロノーム併走録音: 録音画面からメトロノームを起動可(クリック音はマイク越しに混入する旨を注記)
- ストレージ: 合計使用量を表示。`navigator.storage.persist()`を初回に要求(iOSでの自動削除リスクを軽減)

---

## 5.7 練習記録・統計(タブ: ラボ > 「記録」)

### 自動記録
- 各機能の「実練習時間」を自動計測: メトロノーム/進行の再生時間、指板・コード・チューナー画面の操作アクティブ時間(30秒以上無操作は除外)。1日単位・機能別に集計してIndexedDBへ
- 計測はプライバシー上すべて端末内のみ。外部送信なし

### 手動記録
- 「+」からエントリ追加: 日付、練習内容メモ(自由記述)、時間(分)、タグ(基礎練/曲練/理論 等のプリセット+自由追加)

### 統計表示
- 今日の合計 / 今週の日別棒グラフ / 過去12週のカレンダーヒートマップ(色の濃さ=練習時間)
- 連続練習日数(ストリーク)と最長記録
- 機能別内訳(ドーナツまたは横棒)
- 目標: 「1日X分」を設定でき、達成日はヒートマップにリング表示
- グラフは軽量に自前SVG描画(チャートライブラリ不要。rechartsを使う場合はバンドルサイズに注意)

---

## 5.8 設定(タブ: ラボ > 「設定」)

- 左利きモード(§4.1) / チューニング管理(§4.2: プリセット選択・カスタム作成/削除) / 基準ピッチ / 音名表記(§4.3)
- 音量: ギター音・クリック音を個別
- テーマ: 端末に追従 / ダーク / ライト
- 画面を消灯しない(Wake Lock)ON/OFF
- データ: エクスポート / インポート(§4.7)、練習記録の全消去、録音の全消去(いずれも二重確認)
- アプリ情報: バージョン、オフライン対応状況、ホーム画面追加の手順ガイド(初回起動時にも1度だけ案内バナー表示)

## 6. 音楽理論レイヤー仕様(`/src/theory` — UI非依存・要ユニットテスト)

```ts
// ピッチ表現: MIDIノート番号(A4=69)を内部標準とする
type Midi = number;
const REGULAR_TUNING: Midi[] = [40, 45, 50, 55, 59, 64]; // 6弦E2→1弦E4

midiToFreq(midi: Midi, a4: number = 440): number;   // 440 * 2^((m-69)/12)
freqToNearestNote(freq: number, a4: number): { midi: Midi; cents: number };
noteName(midi: Midi, opts: { flat: boolean; solfege: boolean }): string;
fretToMidi(string: 0-5, fret: number, tuning: Midi[]): Midi;
interval(root: Midi, note: Midi): 0-11;              // 度数(半音距離 mod 12)
chordTones(root: 0-11, type: ChordType): Set<0-11>;  // 構成音(半音集合)
scaleTones(root: 0-11, scale: ScaleType): number[];  // インターバル配列から展開
degreeLabel(semitone: 0-11): 'R'|'♭2'|'2'|...|'7';
parseChordSymbol(text: string): { root: 0-11; type: ChordType } | null; // 検索用
transposeVoicing(v: Voicing, semitones: number): Voicing | null;
```

- ChordTypeごとのインターバル定義表(例: maj7=[0,4,7,11], 7♯9=[0,4,7,10,15→mod12=3])を1ファイルに集約
- **テスト必須項目**: 全ChordTypeの構成音 / 全ScaleTypeの展開 / freq⇄note往復誤差 / parseChordSymbolの表記ゆれ(C#m7b5, D♭maj7, Fsus4等30ケース以上) / ボイシングDB全件の構成音一致検証(§5.2)

## 7. iOS Safari固有の制約と対応(実装時の必読事項)

| 制約 | 対応 |
|---|---|
| AudioContextは初回ユーザー操作まで停止 | 共通アンロック処理。全発音経路でresume確認 |
| 消音スイッチで音が出ない場合がある | `navigator.audioSession.type='playback'`(対応時)。非対応なら初回に案内表示 |
| バックグラウンド/画面ロックでタイマーとAudioが停止 | 仕様として許容。再生中はWake Lockで消灯防止。復帰時に再生状態を安全に復元 |
| getUserMediaはHTTPS+ユーザー操作起点が必要 | GitHub PagesはHTTPS。マイク開始は必ずボタン起点 |
| MediaRecorderのMIMEはaudio/mp4が基本 | isTypeSupportedでフォールバック(§5.6) |
| ストレージが自動削除されうる | `navigator.storage.persist()`要求+使用量表示 |
| Vibration API非対応 | ハプティクスは使わず視覚フィードバックで代替 |
| ホーム画面PWAはインストールプロンプト無し | 手動追加の手順ガイドを内蔵(§5.8) |
| 300msタップ遅延/ダブルタップズーム | `touch-action: manipulation`、指板はpointerイベントで自前処理 |
| セーフエリア | viewport-fit=cover + env() |

パフォーマンス目標: 初回ロード(キャッシュ後)1秒以内、JSバンドル(gzip)500KB以下、指板描画60fps、タップ→発音レイテンシ体感50ms以下。

## 8. 実装フェーズ計画(Claude Codeへの発注単位)

1フェーズ=1タスク(1PR)として順に依頼する。各フェーズは「受け入れ基準を満たし、デプロイされた実機URLで確認できる」ことで完了とする。

### Phase 0: 基盤
プロジェクト雛形(Vite+React+TS+PWA)、タブナビゲーション、デザイントークン、設定ストア(左利き/チューニング/表記/テーマ)、AudioEngine骨格(アンロック+Karplus-Strongで1音鳴るデモ)、theory層の主要関数+テスト、GitHub Actions→Pages自動デプロイ。
**受け入れ**: 実機Safariでタブ移動でき、テストボタンで弦の音が鳴り、オフラインで再起動できる。CIでテストが走る。

### Phase 1: メトロノーム(§5.4)
**受け入れ**: BPM/拍子/音価/音色/アクセント編集/タップテンポ/トレーナーが動作。1分間再生してテンポの狂い・音飛びが知覚できない。タブ移動後も再生継続。

### Phase 2: チューナー(§5.5)
**受け入れ**: 実機マイクで6弦すべて検出でき、±5セント判定が安定。音出しモード動作。カスタムチューニング作成→チューナー目標に反映。

### Phase 3: 指板演奏+スケール(§5.1)
**受け入れ**: マルチタッチ発音、左利きミラー、チューニング連動、度数ハイライト、スケールオーバーレイ(全収録スケール)、ポジション絞り込みが動作。

### Phase 4: コードライブラリ(§5.2)
**受け入れ**: 全タイプ×全ルートでボイシングが表示・試聴でき、自動テストで全ボイシングの構成音一致がパス。左利きミラー、検索、お気に入り、「指板で見る」連携が動作。

### Phase 5: コード進行プレイヤー(§5.3)
**受け入れ**: プリセット進行の再生、進行の作成・保存・移調、ストラムパターン切替、現在/次コード表示、クリック併走が動作。

### Phase 6: 録音(§5.6)
**受け入れ**: 実機で録音→保存→再生→A-Bループ→共有→削除が一通り動作。オフラインでも録音・再生可。

### Phase 7: 練習記録+仕上げ(§5.7, §5.8)
**受け入れ**: 自動計測+手動記録+ヒートマップ/グラフ/ストリーク、エクスポート/インポート、全設定項目、初回ガイド。全機能の横断確認(左利き×カスタムチューニング×各機能)。

## 9. Claude Codeでの進め方(発注者向け手順)

1. GitHubアカウントを作成し、空リポジトリ(例: `fretlab`)を作る。この`SPEC.md`をリポジトリ直下に置く
2. iPhoneのブラウザまたはClaudeアプリから claude.ai/code を開き、GitHubを連携してリポジトリを選択
3. ネットワーク設定は「信頼済みソースのみ」(npmインストールに必要)
4. 下のプロンプトでPhase 0から順に依頼し、生成されたPRを確認してマージ→数分後にGitHub PagesのURLを実機Safariで確認→次のPhaseへ
5. 不具合はスクリーンショットとともに「Phase Nの◯◯が仕様§X.Xと異なる」と指摘して修正依頼

### 依頼プロンプト雛形
```
リポジトリ直下の SPEC.md を熟読してから作業してください。
今回のタスクは「Phase N: ◯◎」です。SPEC.md の該当セクション(§…)と
§4 共通仕様、§7 iOS制約を厳守して実装してください。
- 仕様に曖昧さがあれば実装前に解釈を明示すること
- theory/audio 層にはユニットテストを追加し、既存テストを壊さないこと
- 完了時: 変更概要、確認手順(実機での操作手順)、既知の制限を報告すること
```

### 補足
- Claude Codeのモデルは既定のまま(またはSonnet系を指定)でよい。フェーズ分割済みなので1タスクの規模は適正
- Web版Claude Codeはクラウドのサンドボックスで実行されるため、実機確認は必ずデプロイ後のURLで行うこと
